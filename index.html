Node style modules

  • Backbone was so flexible
  • Lacked the power features needed today
  • Evident by everyone having their own Backbone "flavor"
  • Broken up into node-style modules, enhanced for today's features you need

What does node style mean?
  • Everything installed on npm, seriously check out the link
  • Inline with the node/unix philosophy. Check out the linked blog post for further info on this
  • Semver: we're careful to only make breaking changes when necessary, but semver gives us the ability to do that and release quickly

Many people will open an issue or PR and have that issues as a minor or patch release that day

Combine all this built as modules for the browser for easy consumption via browserify or webpack
